# 아티초크버리기: AI 대전

플레이: https://drayage.github.io/Artichoke-discard/

브라우저에서 바로 실행되는 아티초크 덱 제거 카드게임입니다. 2~4명의 로컬 플레이어와 AI를 자유롭게 구성할 수 있으며 기본 모드와 팬 확장 `별난 텃밭`을 지원합니다.

## 실행

별도 빌드 과정은 없습니다. PWA와 오프라인 캐시를 사용하려면 로컬 웹 서버로 실행합니다.

```bash
python -m http.server 5317
```

브라우저에서 `http://127.0.0.1:5317`을 열면 됩니다.

## PWA

- 데스크톱과 모바일 브라우저의 `앱 설치` 또는 `홈 화면에 추가`를 지원합니다.
- 카드 이미지와 게임 파일을 미리 캐시해 최초 접속 이후 오프라인에서도 플레이할 수 있습니다.
- 192px, 512px 마스커블 아이콘과 iOS 홈 화면 아이콘을 포함합니다.

## 구현 내용

- 로컬·AI 혼합 2~4인 턴 진행과 손패 인계 화면
- 플레이어마다 아티초크 10장, 채소 11종마다 6장으로 구성된 정원
- 정원 줄 카드 5장 공개 및 보충
- 카드 사용, 버린 카드 더미, 덱 섞기, 묻은 카드 더미 처리
- 카드 효과에 필요한 손패, 정원 카드 및 상대 선택 UI
- 고정 크기 카드와 수확, 사용, 공개, 교환, 묻기, 뽑기 애니메이션
- 낼 카드가 없을 때 자동 턴 넘김 및 동일 카드만 있는 선택의 자동 처리
- AI 휴리스틱: 아티초크 제거 효과 우선 선택
- 반응형 UI와 게임 로그
- 결과 화면, 배경음과 상황별 효과음
- 모바일 컴팩트 카드와 중복 수량 표시

## 확장 모드

- `변칙 규칙`은 기본 채소 11종을 그대로 사용하고 매 판 무작위 특수 규칙 1개를 적용합니다.
- `별난 텃밭`은 기본 채소 11종과 신규 채소 16종을 합친 27종 전체를 후보로 섞고, 그중 11종만 무작위로 뽑아 정원을 만듭니다. 매 판 무작위 특수 규칙 1개도 적용됩니다.
- `커스텀`은 특수 규칙 1개 또는 규칙 없음을 고르고, 설명을 확인하며 27종 중 사용할 채소 11종을 직접 선택합니다.
- 와사비·오이·방울양배추는 사용자를 제외한 모든 플레이어에게 동시에 예약됩니다. 파슬리가 있는 상대에게는 적용되지 않습니다.
- 파슬리는 새 와사비·오이·방울양배추와 리크·셀러리·치커리처럼 직접 손해를 주는 효과를 무효화합니다. 이득 가능성이 있는 교환·전달 효과와 이미 예약된 방해는 막지 않습니다.
- 심기 카드는 내 앞에 같은 이름을 중복 예약할 수 없지만, 서로 다른 심기 카드는 동시에 예약할 수 있습니다.
- 토마토, 양배추, 순무, 마늘, 버섯, 콜리플라워, 호박, 콩나물, 아스파라거스, 고구마, 와사비, 오이, 방울양배추, 셀러리, 파슬리, 치커리를 포함합니다.
- 심은 카드는 플레이어 현황에 공개되며 다음 차례 시작에 효과를 처리한 뒤 소유자의 버림 더미로 돌아갑니다. 방해 효과도 적용 대상의 현황에 예약 상태로 표시됩니다.
- 풍년, 작은 텃밭, 장날, 돌려짓기, 퇴비 보너스, 품앗이, 큰 바구니, 충동구매 등 14개 판 특수 규칙을 지원합니다.
- `아티초크 대폭발`과 `정리 경쟁`은 즉시 승리 대신 제한 라운드 종료 후 남은 아티초크 수로 순위를 정합니다.

## 턴 순서

1. 보충: 아티초크가 가득한 패로 게임을 시작합니다.
2. 수확: 정원 줄의 앞면 카드 중 1장을 골라 손으로 가져갑니다.
3. 사용: 손에서 내고 싶은 만큼 카드를 냅니다. 카드에 표기된 사항을 모두 이행할 수 있어야 합니다.
4. 버리기: 손에서 내지 않은 카드와 기본 처리되는 낸 카드를 자기 버린 카드 더미로 옮깁니다.
5. 뽑기: 자기 카드 더미에서 카드 5장을 뽑습니다. 카드 더미가 떨어지면 버린 카드 더미를 섞어 새 카드 더미를 만듭니다.

## 승리

자기 차례가 끝나고 5장의 카드를 뽑아 손을 채웠을 때, 뽑은 카드 중 아티초크가 하나도 없다면 즉시 승리합니다.

## 카드

아티초크와 가지, 감자, 당근, 리크, 브로콜리, 비트, 완두콩, 양파, 옥수수, 피망, 프로모 카드 루바브를 구현했습니다.

## 확장 구조

- 플레이어는 고정된 `나/상대`가 아니라 `id`, `seat`, `controller`로 식별합니다.
- 왼쪽 플레이어와 효과 대상은 좌석 순서 및 공용 상대 선택 함수로 결정합니다.
- 연속된 AI 좌석을 자동으로 순회할 수 있어 3~4인 AI 대전으로 확장할 수 있습니다.
- 무작위 처리는 한 함수에 모아 두어 온라인 모드에서는 서버 권위 난수로 교체할 수 있습니다.
- 온라인 모드에서는 `controller`에 원격 플레이어를 추가하고 선택 결과를 서버 액션으로 전달하는 계층이 필요합니다.
